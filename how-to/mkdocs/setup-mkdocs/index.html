    Setting Up MkDocs with Material Theme

    Estimated time to read: 6 minutes

    This guide covers my specific setup. For official documentation, see mkdocs-material.

    Initial Setup

    1. Install Python from python.org

    2. Create and activate a virtual environment: 

      python -m venv venv
source venv/bin/activate  # Linux/Mac
.\venv\Scripts\activate   # Windows

    3. Install required packages: 

      pip install mkdocs
pip install mkdocs-material
pip install mkdocs-git-revision-date-localized-plugin
pip install mkdocs-timetoread-plugin
pip install mkdocs-autolinks-plugin
pip install mkdocs-open-in-new-tab
pip install mkdocs-rss-plugin

      Or use the requirements.txt file: 

      pip install -r requirements.txt

    Configuration

    Site Metadata

    The site includes metadata for SEO and attribution:

    • site_name: Display name for the site
    • site_url: Full URL where the site is hosted
    • site_description: Brief description for search engines
    • site_author: Author attribution
    • repo_url: GitHub repository URL (adds link to repo)
    • repo_name: Display name for repository link
    • copyright: Copyright notice in footer

    Navigation Structure

    The nav section defines the site navigation menu. It supports nested hierarchies and automatically generates the sidebar navigation. See the full structure in the mkdocs.yml file at the root of the repository.

    Full Configuration

    Create or update your mkdocs.yml:

    site_name: "Bryan's knowledge base"
site_url: https://bferg314.github.io
site_description: Technical guides, creative writing, and personal knowledge base covering programming, system administration, and philosophy
site_author: Bryan
repo_url: https://github.com/bferg314/my-knowledge-base
repo_name: bferg314/my-knowledge-base
copyright: Copyright &copy; 2026 Bryan

plugins:
  - search
  - blog:
      blog_toc: true        # Show table of contents
      post_excerpt: optional  # Enable excerpts (use manual or auto)
      post_readtime: true  # Show estimated read time
      pagination_per_page: 5 # Show only 5 posts per page
  - git-revision-date-localized
  - open-in-new-tab:
      add_icon: true
  - autolinks
  - timetoread:
      wpm: 190
      allPages: false
  - tags
  - rss:
      match_path: blog/posts/.*
      date_from_meta:
        as_creation: date
      categories:
        - categories
        - tags

theme:
  name: material
  favicon: assets/favicon.png
  icon:
    logo: material/book-open-page-variant
  features:
    - content.code.copy
    - content.code.select
    - content.code.annotate
  palette:
    # Palette toggle for automatic mode
    - media: "(prefers-color-scheme)"
      toggle:
        icon: material/brightness-auto
        name: Switch to light mode

    # Palette toggle for light mode
    - media: "(prefers-color-scheme: light)"
      scheme: default
      toggle:
        icon: material/brightness-7
        name: Switch to dark mode

    # Palette toggle for dark mode
    - media: "(prefers-color-scheme: dark)"
      scheme: slate
      toggle:
        icon: material/brightness-4
        name: Switch to system preference

markdown_extensions:
  - pymdownx.highlight:
      anchor_linenums: true
      line_spans: __span
      pygments_lang_class: true
  - pymdownx.inlinehilite
  - pymdownx.snippets
  - admonition
  - pymdownx.details
  - pymdownx.superfences:
      custom_fences:
        - name: mermaid
          class: mermaid
          format: !!python/name:pymdownx.superfences.fence_code_format
  - pymdownx.tabbed:
      alternate_style: true
  - md_in_html

extra_css:
  - assets/stylesheets/print.css

    Advanced Features

    Material Theme Admonitions

    Create callout boxes using the following syntax:

    !!! note "Optional Title"
    Note content here

!!! warning
    Warning content here

!!! tip
    Tip content here

    Tabbed Content

    Create tabbed sections using:

    === "Tab 1"
    Content for tab 1

=== "Tab 2"
    Content for tab 2

    Mermaid Diagrams

    Create diagrams using Mermaid syntax:

    ```mermaid
graph LR
    A[Start] --> B{Decision}
    B -->|Yes| C[Process]
    B -->|No| D[End]
    C --> D
```

    Code Blocks

    Code blocks support:

    • Syntax highlighting
    • Line numbers
    • Copy button
    • Code selection
    • Annotations

    Example: 

    def hello_world():  # (1)
    print("Hello")  # (2)

# 1. Function definition
# 2. Print statement

    Blog Features

    The blog plugin provides:

    • Table of contents for blog posts
    • Optional excerpts (manual or automatic)
    • Estimated read time
    • Pagination (5 posts per page)
    • RSS feed for blog posts

    RSS Feed

    The RSS plugin generates a feed for blog posts that match blog/posts/.*, including post dates, categories, and tags.

    Building and Deploying

    1. Local Development: 

      mkdocs serve
      Navigate to http://127.0.0.1:8000 to preview your site.

    2. Build Static Site: 

      mkdocs build --clean
      The static site will be generated in the site/ directory.

    3. Deploy to GitHub Pages:

      Automated Deployment (Recommended): This repository uses GitHub Actions to automatically deploy to GitHub Pages when pushing to the main branch. The workflow:

      • Builds the site using mkdocs build --clean
      • Deploys to the bferg314.github.io repository
      • Requires a GH_PAT secret with appropriate permissions

      See the .github/workflows/deploy.yml file in the repository root for the full configuration.

      Manual Deployment (Windows): 

      mkdocs build --clean
robocopy "site" "../bferg314.github.io" /MIR /XD ".git"
cd ../bferg314.github.io
git add .
git commit -m "Update documentation"
git push

    GitHub Actions Setup

    The automated deployment workflow requires:

    1. GitHub Personal Access Token (PAT):

      • Create a PAT with repo permissions
      • Add it as a secret named GH_PAT in the repository settings

    2. Workflow Configuration:

      • Located at .github/workflows/deploy.yml in the repository root
      • Triggers on push to main branch
      • Can be manually triggered via workflow_dispatch

    3. Deployment Process:

      • Checks out the source repository
      • Installs Python dependencies from requirements.txt
      • Builds the site with mkdocs build --clean
      • Deploys to the bferg314.github.io repository

    Useful Tips

    Front Matter

    Always include front matter in your markdown files: 

    ---
title: Page Title
description: Page description
tags:
  - tag1
  - tag2
timetoread: True
---

    Custom Branding

    Customize your site's appearance: - Add a favicon: Place favicon.png in docs/assets/ - Set a custom logo icon using Material icons: icon.logo: material/book-open-page-variant - Add custom CSS in docs/assets/stylesheets/

    Blog Posts

    For blog posts in docs/blog/posts/: - Use .md extension (e.g., 2026-02-01.md) - Include date in front matter for RSS feed - Posts are automatically sorted by date - Pagination shows 5 posts per page

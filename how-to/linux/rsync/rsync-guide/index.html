    RSync Guide

    What is rsync?

    rsync (remote sync) is a powerful and versatile command-line utility in Linux used for efficiently transferring and synchronizing files and directories between two locations. It is widely used for backups, mirroring, and general file synchronization.

    Unlike cp, rsync is designed to minimize data transfer by only copying the differences between the source and destination, making it faster and more efficient, especially for large datasets or over networks.

    Key Features of rsync

    Key Features

    • Efficient File Transfer: Only transfers the parts of files that have changed
    • Preserves File Attributes: Maintains permissions, timestamps, symbolic links, and ownership
    • Supports Remote Transfers: Works with both local and remote systems via SSH
    • Recursive Directory Copying: Handles entire directory structures
    • Partial Transfers: Can resume interrupted transfers
    • Dry Run Mode: Test runs without making changes
    • Compression: Reduces bandwidth usage during transfers

    Basic Syntax

    rsync [options] source destination

    Tip

    • source: The file or directory to copy
    • destination: The location where the file or directory will be copied to
    • Remember: trailing slash matters! /source/ copies contents, /source copies the directory itself

    Common Use Cases and Examples

    1. Copy Files Locally

    rsync -av /path/to/source/ /path/to/destination/

    Common Options Explained

    • -a: Archive mode (preserves permissions, timestamps, symbolic links, etc.)
    • -v: Verbose mode (shows progress and details)

    Note: The trailing / in /path/to/source/ means "copy the contents of the directory." Without the /, it copies the directory itself.

    2. Copy Files to a Remote Server

    rsync -avz /path/to/source/ user@remote:/path/to/destination/
    - -z: Compresses data during transfer. - user@remote: Specifies the remote user and server address.

    3. Copy Files from a Remote Server

    rsync -avz user@remote:/path/to/source/ /path/to/destination/

    4. Dry Run (Test Without Copying)

    rsync -av --dry-run /path/to/source/ /path/to/destination/
    - This shows what rsync would do without making any changes.

    5. Delete Files in Destination That Don’t Exist in Source

    rsync -av --delete /path/to/source/ /path/to/destination/
    - This ensures the destination is an exact mirror of the source.

    6. Resume Interrupted Transfers

    rsync -av --partial /path/to/source/ /path/to/destination/
    - --partial: Keeps partially transferred files, allowing the transfer to resume later.

    7. Limit Bandwidth Usage

    rsync -av --bwlimit=1000 /path/to/source/ /path/to/destination/
    - --bwlimit=1000: Limits bandwidth usage to 1000 KB/s.

    8. Exclude Files or Directories

    rsync -av --exclude='*.log' /path/to/source/ /path/to/destination/
    - --exclude='*.log': Excludes all .log files from the transfer.

    9. Include Only Specific Files

    rsync -av --include='*.txt' --exclude='*' /path/to/source/ /path/to/destination/
    - This copies only .txt files and excludes everything else.

    10. Preserve Ownership and Group (Requires Root)

    sudo rsync -av --owner --group /path/to/source/ /path/to/destination/

    Common Options

    Option Description
    -a Archive mode (preserves permissions, timestamps, symbolic links, etc.).
    -v Verbose mode (shows progress and details).
    -z Compresses data during transfer.
    -r Recursively copies directories.
    --delete Deletes files in the destination that don’t exist in the source.
    --dry-run Simulates the operation without making any changes.
    --partial Keeps partially transferred files for resuming later.
    --progress Shows progress during the transfer.
    --bwlimit Limits bandwidth usage (in KB/s).
    --exclude Excludes files or directories matching a pattern.
    --include Includes only files or directories matching a pattern.
    -e ssh Specifies SSH as the remote shell (default for remote transfers).

    Advanced Usage

    1. Using SSH with a Specific Port

    rsync -avz -e 'ssh -p 2222' /path/to/source/ user@remote:/path/to/destination/
    - -e 'ssh -p 2222': Uses SSH on port 2222.

    2. Syncing Large Datasets

    rsync -avz --progress /path/to/large_dataset/ user@remote:/path/to/destination/
    - --progress: Shows progress for each file.

    3. Backing Up to a Remote Server

    rsync -avz --backup --backup-dir=/backup/old_files /path/to/source/ user@remote:/path/to/destination/
    - --backup: Creates backups of files that are overwritten. - --backup-dir: Stores backups in a separate directory.

    Why Use rsync Over cp?

    Advantages

    • Efficiency: Only transfers changes, saving time and bandwidth
    • Flexibility: Works locally and remotely
    • Resumable Transfers: Can resume interrupted transfers
    • Advanced Options: Supports exclusions, bandwidth limiting, and more

    Pro Tip

    rsync is an indispensable tool for system administrators and users who need to manage file transfers and backups efficiently. Its versatility and performance make it a go-to solution for many file synchronization tasks.

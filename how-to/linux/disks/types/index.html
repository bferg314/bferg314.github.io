    Linux Filesystem Types

    Overview

    Linux supports multiple filesystem types, each optimized for specific use cases. This guide covers the most common options and their ideal applications.

    Native Linux Filesystems

    ext4 (Fourth Extended Filesystem)

    Default Choice

    ext4 is the default filesystem for most Linux distributions, offering a good balance of features, performance, and reliability.

    • Journaling support
    • Large file/volume support
    • Delayed allocation
    • Extents
    • Desktop systems
    • Laptops
    • General-purpose servers
    • System partitions
    sudo mkfs.ext4 /dev/sdX#

    # Basic mount
sudo mount -t ext4 /dev/sdX# /mount/point

# Common fstab options
/dev/sdX#  /mount/point  ext4  defaults,noatime,errors=remount-ro  0  2

# Performance options
/dev/sdX#  /mount/point  ext4  defaults,noatime,data=writeback,barrier=0  0  2

    Common Options

    • noatime: Disable access time updates
    • errors=remount-ro: Remount read-only on errors
    • data=writeback: Faster performance but less safe
    • barrier=0: Disable barriers (use with caution)

    XFS

    Enterprise Choice

    XFS excels in environments requiring high performance and large file handling.

    • High performance
    • Online defragmentation
    • Quota support
    • Parallel I/O operations
    • Enterprise servers
    • Large storage systems
    • High-performance workloads
    sudo mkfs.xfs /dev/sdX#

    # Basic mount
sudo mount -t xfs /dev/sdX# /mount/point

# Common fstab options
/dev/sdX#  /mount/point  xfs  defaults,noatime,allocsize=64m  0  2

# Performance options
/dev/sdX#  /mount/point  xfs  defaults,noatime,largeio,swalloc  0  2

    Common Options

    • allocsize=64m: Allocation size for optimal performance
    • largeio: Enable large I/O operations
    • swalloc: Stripe-width allocation for RAID

    Btrfs

    Modern Features

    Btrfs offers advanced features like snapshots and self-healing, making it ideal for modern systems.

    • Copy-on-write
    • Snapshots
    • Built-in RAID support
    • Data integrity checks
    • Subvolumes
    • Advanced storage setups
    • Systems requiring snapshots
    • Software RAID configurations
    sudo mkfs.btrfs /dev/sdX#

    # Basic mount
sudo mount -t btrfs /dev/sdX# /mount/point

# Common fstab options
/dev/sdX#  /mount/point  btrfs  defaults,noatime,compress=zstd  0  0

# Subvolume mount
/dev/sdX#  /mount/point  btrfs  defaults,subvol=@home,compress=zstd  0  0

    Common Options

    • compress=zstd: Enable compression
    • subvol=@name: Mount specific subvolume
    • ssd: Enable SSD optimizations
    • autodefrag: Enable automatic defragmentation

    ZFS

    License Note

    ZFS isn't included in the Linux kernel due to licensing issues. Additional setup may be required.

    • Pooled storage
    • Data integrity
    • Snapshots
    • Deduplication
    • Compression
    • Enterprise storage
    • Data centers
    • NAS systems
    sudo zpool create mypool /dev/sdX#

    # Import pool
sudo zpool import mypool

# Mount dataset
sudo zfs mount mypool/dataset

# Automatic mount in /etc/zfs/zfs-list.cache/
mypool/dataset  /mount/point  zfs  defaults,atime=off,compression=on  0  0

    Common Options

    • compression=on: Enable compression
    • atime=off: Disable access time updates
    • recordsize=128k: Optimize for large files
    • xattr=sa: Store extended attributes in inodes

    Windows-Compatible Filesystems

    Cross-Platform Compatibility

    These filesystems allow data sharing between Linux and Windows systems.

    NTFS

    • Windows native filesystem
    • Large file support
    • Permissions support
    • Journaling
    # Install NTFS support
sudo apt install ntfs-3g  # Debian/Ubuntu
sudo dnf install ntfs-3g  # Fedora
sudo pacman -S ntfs-3g    # Arch

# Format command
sudo mkfs.ntfs /dev/sdX#

    # Basic mount
sudo mount -t ntfs-3g /dev/sdX# /mount/point

# Common fstab options
/dev/sdX#  /mount/point  ntfs-3g  defaults,uid=1000,gid=1000,dmask=022,fmask=133  0  0

# Windows compatibility
/dev/sdX#  /mount/point  ntfs-3g  defaults,windows_names,hide_hid_files  0  0

    Common Options

    • uid=1000,gid=1000: Set owner
    • dmask=022,fmask=133: Set permissions
    • windows_names: Enable Windows filename rules
    • big_writes: Improve write performance

    FAT32

    • Universal compatibility
    • Simple structure
    • 4GB file size limit
    sudo mkfs.vfat -F 32 /dev/sdX#

    # Basic mount
sudo mount -t vfat /dev/sdX# /mount/point

# Common fstab options
/dev/sdX#  /mount/point  vfat  defaults,uid=1000,gid=1000,umask=022  0  0

# UTF-8 support
/dev/sdX#  /mount/point  vfat  defaults,iocharset=utf8,codepage=437  0  0

    Common Options

    • uid=1000,gid=1000: Set owner
    • umask=022: Set permissions
    • iocharset=utf8: Enable UTF-8 support
    • codepage=437: Set character encoding

    exFAT

    • Large file support
    • No journaling
    • Cross-platform compatible
    # Install exFAT support
sudo apt install exfat-fuse exfat-utils  # Debian/Ubuntu
sudo dnf install exfatprogs              # Fedora
sudo pacman -S exfatprogs               # Arch

# Format command
sudo mkfs.exfat /dev/sdX#

    # Basic mount
sudo mount -t exfat /dev/sdX# /mount/point

# Common fstab options
/dev/sdX#  /mount/point  exfat  defaults,uid=1000,gid=1000,umask=022  0  0

# Performance options
/dev/sdX#  /mount/point  exfat  defaults,max_read=1M,cache=normal  0  0

    Common Options

    • uid=1000,gid=1000: Set owner
    • umask=022: Set permissions
    • max_read=1M: Set maximum read size
    • cache=normal: Enable caching

    Important Notes

    • Replace sdX# with your actual device/partition (e.g., sda1, nvme0n1p1)
    • Always double-check device names before formatting
    • Back up important data before filesystem operations

    Best Practices

    Filesystem Selection

    1. System Partition: Use ext4 for reliability
    2. Large Files: Consider XFS or ZFS
    3. Snapshots/RAID: Choose Btrfs or ZFS
    4. Cross-Platform: Use exFAT for external drives
    5. Legacy Systems: FAT32 for universal compatibility

    Troubleshooting

    Common Issues

    • Mount Errors: Check filesystem type and mount options
    • Performance Issues: Consider filesystem optimization tools
    • Corruption: Use appropriate filesystem check tools (e.g., fsck, xfs_repair)

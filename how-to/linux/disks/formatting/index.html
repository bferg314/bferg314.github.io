    Disk Formatting in Linux

    Data Loss Warning

    • All commands in this guide can result in permanent data loss
    • Always backup important data before proceeding
    • Double-check device names (e.g., nvme0n1, sda)
    • Commands require root privileges
    • If unsure, consult a system administrator

    Identifying Your Disk

    1. List all block devices: 

      lsblk

      Example output: 

      NAME        MAJ:MIN RM   SIZE RO TYPE MOUNTPOINT
sda           8:0    0 931.5G  0 disk
└─sda1        8:1    0 931.5G  0 part /
nvme0n1     259:0    0 476.9G  0 disk

    Device Naming

    • sdX: SATA/USB drives
    • nvmeXnY: NVMe drives
    • mmcblkX: SD cards/eMMC storage

    Partitioning Tools

    Features

    • Terminal UI with menu-driven interface
    • Supports both MBR and GPT
    • Easier to use than fdisk/gdisk
    • Automatic partition alignment
    sudo cfdisk /dev/sdX

    Navigation:

    • Arrow keys - move selection
    • Enter - select option
    • [New] - create partition
    • [Delete] - remove partition
    • [Type] - change partition type
    • [Write] - save changes
    • [Quit] - exit

    Usage Tips

    • Space bar selects bootable flag
    • Select [Write] then type "yes" to confirm changes
    • Always [Quit] properly to ensure changes are saved

    Features

    • Traditional MBR partitioning
    • Up to 2TB disk size
    • Maximum 4 primary partitions
    • Legacy BIOS systems
    sudo fdisk /dev/sdX

    Common commands:

    • d - delete partition
    • n - new partition
    • p - print partition table
    • w - write changes
    • q - quit without saving

    Features

    • Modern GPT partitioning
    • Unlimited disk size
    • Up to 128 partitions
    • Required for UEFI
    • Includes backup partition table
    sudo gdisk /dev/sdX

    Common commands:

    • n - new partition
    • d - delete partition
    • l - list known types
    • t - change type
    • w - write changes
    • q - quit

    Choosing a Partitioning Tool

    • cfdisk: Easiest to use, good for beginners
    • fdisk: Traditional CLI tool, great for scripting
    • gdisk: Modern GPT-focused tool with advanced features

    Partition Types

    Common GPT Partition Types

    8300 - Linux filesystem
8200 - Linux swap
ef00 - EFI System
fd00 - Linux RAID

    Creating Filesystems

    sudo mkfs.ext4 /dev/sdX1

# With label
sudo mkfs.ext4 -L mydata /dev/sdX1

    sudo mkfs.btrfs /dev/sdX1

# With compression
sudo mkfs.btrfs -L mydata --compress zstd /dev/sdX1

    sudo mkfs.xfs /dev/sdX1

# With label
sudo mkfs.xfs -L mydata /dev/sdX1

    # Install tools first
sudo apt install exfat-fuse exfat-utils     # Debian/Ubuntu
sudo dnf install exfat-utils fuse-exfat     # Fedora
sudo pacman -S exfatprogs                   # Arch

# Format
sudo mkfs.exfat -n mydata /dev/sdX1

    Mounting

    Temporary Mount

    sudo mkdir -p /mnt/mydisk
sudo mount /dev/sdX1 /mnt/mydisk

    Permanent Mount

    1. Get partition UUID: 

      sudo blkid /dev/sdX1

    2. Edit /etc/fstab: 

      # Add line:
UUID=your-uuid-here  /mount/point  filesystem  options  0  2

    3. Applying fstab Changes Immediately

      To apply changes made to /etc/fstab without rebooting, use the following command:

      sudo mount -a

      This command mounts all filesystems mentioned in /etc/fstab that are not currently mounted.

    UUID=1234-5678 /mnt/data ext4 defaults,noatime 0 2

    UUID=1234-5678 /mnt/data btrfs defaults,compress=zstd 0 0

    UUID=1234-5678 /mnt/data exfat uid=1000,gid=1000,dmask=022,fmask=133 0 0

    Post-Mount Setup

    Setting Permissions

    # Single user
sudo chown -R $USER:$USER /mnt/mydisk

# Group access
sudo chown -R :groupname /mnt/mydisk
sudo chmod -R 775 /mnt/mydisk

    Quick Reference

    Common Commands

    # List disks
lsblk
fdisk -l

# Check filesystem
fsck /dev/sdX1

# Show UUID
blkid

# Show mount points
df -h
findmnt

    Troubleshooting

    Common Issues

    • Mount Failed: Check filesystem type and mount options
    • Permission Denied: Verify ownership and permissions
    • Disk Not Found: Check if disk is recognized in dmesg
    • Cannot Format: Ensure partition isn't mounted

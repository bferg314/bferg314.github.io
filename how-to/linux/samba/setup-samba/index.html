    Samba File Sharing Setup

    Overview

    Samba allows you to share files and folders across your network, making them accessible to Windows, Linux, and macOS systems.

    Installation

    sudo apt update
sudo apt install samba samba-common-bin

    sudo dnf install samba samba-common samba-client

    sudo pacman -S samba

    Basic Configuration

    Create Share Directory

    sudo mkdir /share
sudo chmod 777 /share

    Permissions

    The 777 permissions are not recommended for production. Consider using more restrictive permissions based on your needs.

    Configure Samba

    1. Backup existing config: 

      sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.bak

    2. Edit configuration: 

      [global]
workgroup = WORKGROUP
security = user
map to guest = bad user

[share]
path = /share
browseable = yes
read only = no
force create mode = 0660
force directory mode = 0770
public = yes

    3. Apply configuration: 

      # Test configuration syntax
testparm

# Restart Samba to apply changes
sudo systemctl restart smbd nmbd

    Configuration Options

    • workgroup: Match your Windows workgroup
    • security: Authentication method
    • map to guest: Handle unknown users
    • force create mode: Default file permissions
    • force directory mode: Default folder permissions

    User Management

    sudo smbpasswd -a $USER

    sudo pdbedit -L -v

    sudo smbpasswd -x username

    Service Management

    # Start services
sudo systemctl start smbd nmbd

# Enable at boot
sudo systemctl enable smbd nmbd

# Check status
sudo systemctl status smbd

    Firewall Configuration

    sudo ufw allow samba

    sudo firewall-cmd --permanent --add-service=samba
sudo firewall-cmd --reload

    sudo iptables -A INPUT -p tcp -m tcp --dport 139 -j ACCEPT
sudo iptables -A INPUT -p tcp -m tcp --dport 445 -j ACCEPT
sudo iptables -A INPUT -p udp -m udp --dport 137 -j ACCEPT
sudo iptables -A INPUT -p udp -m udp --dport 138 -j ACCEPT

    Accessing Shares

    Connection Examples

    File Explorer: \\server-ip\share

    File Manager: smb://server-ip/share
Mount Command: mount -t cifs //server-ip/share /mnt/point

    Finder: smb://server-ip/share

    Advanced Configuration

    Security Enhancements

    [share]
valid users = @users
guest ok = no
writable = yes
browseable = yes

    Performance Tuning

    [global]
socket options = TCP_NODELAY IPTOS_LOWDELAY
write cache size = 262144

    Troubleshooting

    Common Issues

    1. Permission Denied 

      # Check SELinux (Fedora)
sudo setsebool -P samba_enable_home_dirs on

# Check directory permissions
ls -l /share

    2. Connection Failed 

      # Test configuration
testparm

# Check logs
sudo tail -f /var/log/samba/log.smbd

    3. Service Issues 

      # Restart services
sudo systemctl restart smbd nmbd

# Check ports
sudo netstat -tulpn | grep smb

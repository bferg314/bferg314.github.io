    Vim: A Quick Start Guide

    About Vim

    Vim is a highly efficient text editor that uses keyboard-based commands and different modes to edit text. While its learning curve can be steep, the productivity gains are substantial.

    Core Concepts

    Understanding Modes

    Vim's power comes from its modal editing system. Think of modes as different "states" where keys behave differently:

    • Normal Mode (🔍): For navigation and commands
    • Insert Mode (✏️): For typing text
    • Visual Mode (📑): For selecting and manipulating text
    • Command Mode (:): For executing commands

    Mode Transitions

    graph TD N[Normal Mode] I[Insert Mode] V[Visual Mode] C[Command Mode] N -->|"i, a, I, A"| I I -->|Esc| N N -->|"v, V, Ctrl+v"| V V -->|Esc| N N -->|:| C C -->|Enter| N

    Essential Commands

    Must-Know Commands

    • h/j/k/l - Left/Down/Up/Right
    • w/b - Next/previous word
    • 0/$ - Start/end of line
    • gg/G - First/last line
    • :+number - Jump to line number
    • i/a - Insert before/after cursor
    • o/O - New line below/above
    • x - Delete character
    • dd - Delete line
    • yy - Copy line
    • p - Paste after cursor
    • /pattern - Search forward
    • ?pattern - Search backward
    • n/N - Next/previous match
    • :%s/old/new/g - Replace all

    Common Operations

    Practical Examples

    # Delete inside parentheses
di(

# Change inside quotes
ci"

# Copy to end of line
y$

# Delete 3 words
d3w

# Change until semicolon
ct;

# Select all text in file
ggVG

# Delete from cursor to end of file
dG

# Copy current paragraph
yip

# Change inside function (between braces)
ci{

# Delete everything inside HTML tag
dit

# Replace current character with 'x'
rx

# Delete from cursor to next occurrence of 'foo'
d/foo

# Copy from cursor to end of paragraph
y}

# Change until next closing parenthesis
ct)

# Delete all lines containing 'error'
:g/error/d

# Indent inside current block
>i{

    Understanding the Examples

    • gg jumps to start, V enters line visual mode, G jumps to end
    • ip means "inner paragraph"
    • it means "inner tag"
    • ci{ changes everything inside curly braces
    • :g performs actions on lines matching a pattern

    Power User Tips

    Quick Tricks

    • Use . to repeat last change
    • % jumps between matching brackets
    • * searches for word under cursor
    • zz centers current line
    • Ctrl+v + I for block insert

    Text Objects

    Text Object Reference

    Combine with operators (d, c, y):

    • w - word
    • s - sentence
    • p - paragraph
    • t - tag
    • " - quoted text
    • () or {} - parentheses/braces

    Advanced Features

    Beyond Basics

    • ma - Set mark 'a'
    • `a - Jump to mark 'a'
    • '. - Jump to last change
    • "ay - Yank to register 'a'
    • "ap - Paste from register 'a'
    • :reg - View registers
    • qa - Record to register 'a'
    • q - Stop recording
    • @a - Play macro 'a'
    • "+y - Copy to system clipboard
    • "+p - Paste from system clipboard
    • :set clipboard=unnamedplus - Use system clipboard by default

    Clipboard Support

    Ensure Vim has clipboard support with: 

    vim --version | grep clipboard
    Install vim-gtk or gvim if needed.

    Configuration

    Vim Setup

    1. Create/edit ~/.vimrc
    2. Common settings: 
      set number          " Line numbers
set relativenumber  " Relative line numbers
set expandtab       " Spaces instead of tabs
set shiftwidth=4    " Indentation size
set hidden          " Buffer handling
set ignorecase      " Case-insensitive search
set smartcase       " Smart case search

    Learning Resources

    Further Learning

    1. Run vimtutor in terminal
    2. Visit Vim Tips Wiki
    3. Try vim-adventures.com
    4. Use :help within Vim

    Practice Makes Perfect

    The key to mastering Vim is regular practice. Start with basic commands and gradually incorporate new ones into your workflow. Use Ctrl+[ as an alternative to Esc.

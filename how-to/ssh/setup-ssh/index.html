    Creating and Using SSH Keys

    Windows-Specific Notes

    • Most commands can be run in PowerShell, but ensure you have OpenSSH installed
    • Windows uses backslashes (\) in paths, but use forward slashes (/) in WSL and Git Bash
    • Some commands may require running PowerShell as administrator

    Creating an SSH Key

    1. Open any Linux Terminal, PowerShell, or Windows Terminal

    2. Generate a new SSH key pair: 

      ssh-keygen -t ed25519 -C "your_email@example.com"

    3. When prompted, press Enter to accept the default path (~/.ssh/id_ed25519) or specify a custom path

    4. Optionally set a passphrase when prompted

    5. Verify your keys in the .ssh directory:

      • Public key: ~/.ssh/id_ed25519.pub
      • Private key: ~/.ssh/id_ed25519

    Why ed25519?

    • Enhanced Security: Stronger cryptography with smaller keys than RSA
    • Better Performance: Faster key operations than older algorithms
    • Compact: Smaller key sizes ideal for storage
    • Modern Design: Built on elliptic curve cryptography

    Adding the SSH Key to the Agent

    Windows Setup

    PowerShell (As Administrator)

    Set-Service -Name ssh-agent -StartupType Automatic
Start-Service ssh-agent

    Linux Setup

    eval "$(ssh-agent -s)"

    Adding Your Key

    ssh-add ~/.ssh/id_ed25519

    ssh-add $env:USERPROFILE/.ssh/id_ed25519

    Verify the key is loaded: 

    ssh-add -l

    Using SSH Keys for Remote Connections

    Adding Your Key to a Remote Host

    ssh-copy-id user@remote-host
    1. Copy your public key: 
      cat ~/.ssh/id_ed25519.pub
    2. On the remote server: 
      mkdir -p ~/.ssh
echo "PASTE_YOUR_PUBLIC_KEY_HERE" >> ~/.ssh/authorized_keys
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

    Test your connection: 

    ssh user@remote-host

    GitHub Configuration

    1. Copy your public key: 

      cat ~/.ssh/id_ed25519.pub

    2. Add it to GitHub:

      • Go to Settings > SSH and GPG keys
      • Click New SSH Key
      • Paste and save

    3. Test GitHub connection: 

      ssh -T git@github.com

    4. Configure Git repository: 

      git remote set-url origin git@github.com:username/repository.git

    WSL Integration

    1. Link Windows SSH directory: 

      ln -s /mnt/c/Users/<YourUsername>/.ssh ~/.ssh

    2. Set proper permissions: 

      chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
chmod 600 ~/.ssh/authorized_keys

    Permission Requirements

    SSH requires strict file permissions for security:

    • 700 for .ssh directory: only you can access
    • 600 for key files: only you can read/write
    1. Configure WSL SSH agent: 
      eval $(ssh-agent)
ssh-add ~/.ssh/id_ed25519

    Best Practices

    SSH Configuration

    Create ~/.ssh/config to manage multiple keys: 

    Host github.com
    User git
    IdentityFile ~/.ssh/id_ed25519

    Security Tips

    • Back up your private key securely
    • Use different keys for different purposes
    • Always protect private keys with strong permissions
    • Consider using a passphrase for sensitive keys

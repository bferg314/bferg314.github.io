    Chocolatey Guide

    Prerequisites

    Note

    Before installing Chocolatey, you need to set the execution policy for PowerShell scripts. Open PowerShell as an administrator and run the following command:

    Set-ExecutionPolicy RemoteSigned

    Installing Chocolatey

    To install Chocolatey, open an elevated PowerShell (Run as Administrator) and run the following command:

    Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

    Warning

    Ensure you have administrative privileges when running the PowerShell commands.

    Main Commands

    Installing a Package

    To install a package, use the choco install command followed by the package name. For example, to install Google Chrome:

    choco install googlechrome

    You can install multiple packages at once by separating them with spaces:

    choco install googlechrome firefox 7zip

    Upgrading Packages

    To upgrade all installed packages to their latest versions, use:

    choco upgrade all

    To upgrade a specific package:

    choco upgrade googlechrome

    Uninstalling Packages

    To uninstall a package, use the choco uninstall command followed by the package name:

    choco uninstall googlechrome

    Listing Installed Packages

    To list all installed packages, use:

    choco list

    Searching for Packages

    To search for a package, use the choco search command followed by the package name or a keyword:

    choco search chrome

    Viewing Package Information

    To view detailed information about a package, use the choco info command followed by the package name:

    choco info googlechrome

    Installing Specific Versions

    To install a specific version of a package, use the --version flag:

    choco install googlechrome --version=91.0.4472.124

    Creating a Package

    To create a new Chocolatey package, use the choco new command followed by the package name:

    choco new mypackage

    This will create a folder with the necessary files to build your package. Edit the .nuspec file to define your package's metadata.

    Exporting and Importing Packages

    To export a list of installed packages to a file:

    choco export --output-file=packages.config

    To install packages from a file:

    choco install packages.config

    Setting Up a Local Repository

    To set up a local repository, create a folder to store your packages and configure Chocolatey to use it:

    choco source add -n=local -s="C:\path\to\your\repository"

    You can now add packages to this folder and install them using Chocolatey.

    Using Chocolatey GUI

    For a more user-friendly experience, you can install Chocolatey GUI:

    choco install chocolateygui

    This provides a graphical interface for managing your Chocolatey packages.

    Tip

    Chocolatey GUI can be especially helpful for users who prefer a visual interface over command-line operations.

    Additional Resources

    For more information and advanced usage, refer to the Chocolatey documentation.

    Conclusion

    These tips and tricks should help you get started with Chocolatey and make the most of its features. Happy package managing!

    Installing Docker on Linux

    Distribution Support

    This guide covers Docker installation on:

    • Debian/Ubuntu-based distributions
    • Fedora/RHEL-based distributions
    • Arch Linux and derivatives

    Prerequisites

    System Requirements

    • 64-bit Linux distribution
    • Kernel version 3.10 or newer
    • SystemD as init system
    • At least 4GB RAM

    Installation Steps

    1. Update package index and install prerequisites: 

      sudo apt update
sudo apt install -y ca-certificates curl gnupg lsb-release

    2. Add Docker's official GPG key: 

      sudo mkdir -m 0755 -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

    3. Set up repository: 

      echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

    4. Install Docker Engine: 

      sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

    1. Add Docker repository: 

      sudo dnf -y install dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo

    2. Install Docker Engine: 

      sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

    1. Install Docker packages: 
      sudo pacman -Syu
sudo pacman -S docker docker-compose

    Post-Installation Setup

    1. Start and enable Docker service: 

      sudo systemctl start docker
sudo systemctl enable docker

    2. Add your user to the docker group: 

      sudo usermod -aG docker $USER

    Group Changes

    Log out and back in for group changes to take effect.

    1. Verify installation: 
      docker --version
docker run hello-world

    Troubleshooting

    Common Issues

    • Permission Denied: Ensure your user is in the docker group
    • Service Not Starting: Check system logs with journalctl -xu docker
    • Network Issues: Verify Docker network settings with docker network ls

    Security Recommendations

    Security Best Practices

    • Regularly update Docker and containers
    • Use rootless containers when possible
    • Implement resource limits
    • Scan images for vulnerabilities
    • Use content trust for image verification

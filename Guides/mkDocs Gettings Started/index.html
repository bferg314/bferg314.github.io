    Getting Started with MkDocs: A Simple Guide

    Step 1: Installing MkDocs

    MkDocs is a great tool for making simple websites out of Markdown files. Here’s how you can get it set up:

    1. Open Your Command Line: You can use Command Prompt or PowerShell. If you're on a Mac, open Terminal.
    2. Install Python (if you don't already have it): MkDocs requires Python to run. Head to python.org and install the latest version.
    3. Install MkDocs:
    4. Once Python is installed, use this command to install MkDocs: 
      pip install mkdocs
    5. You should also install the Material theme for a great look: 
      pip install mkdocs-material

    Step 2: Create Your GitHub Pages Repository

    To host your site online, you’ll need a GitHub Pages repository.

    1. Create Your GitHub Repository:

    2. Go to GitHub and create a new repository called my-knowledge-base.

    3. Leave the repository empty — no README or other files.

    4. Clone Your Repository:

    5. Go to your terminal and type: 
      git clone https://github.com/yourusername/my-knowledge-base.git
    6. Replace yourusername with your GitHub username.
    7. This will create a folder called my-knowledge-base where you can put your files.

    Step 3: Setting Up Your MkDocs Site

    Now, let’s create a simple MkDocs site to get you started.

    1. Navigate to Your Repository Directory:
    cd my-knowledge-base

    1. Create a New MkDocs Project:

    2. Run this command to create a new MkDocs project: 

      mkdocs new .

    3. This will create the files you need, including mkdocs.yml and a folder called docs where you’ll put all your Markdown files.

    4. Customize Your Site (optional):

    5. Open mkdocs.yml in your favorite editor.
    6. Update the site name and theme: 
      site_name: "My Knowledge Base"
site_url: https://yourrepository.github.io
theme:
  name: material

    Step 4: Serve and Test Your Site Locally

    Before publishing it online, let’s see what it looks like!

    1. Serve the Site:
    2. Run the following command to see your site in your browser: 
      mkdocs serve
    3. Open your web browser and go to http://127.0.0.1:8000. You should see your site with the default page!

    Step 5: Build the Static Files

    You need to build your MkDocs site into static files to publish it online.

    1. Build Your Site:
    2. Run this command to generate the static HTML files: 
      mkdocs build
    3. This will create a folder called site/ with everything you need to publish your site.

    Step 6: Create a Separate Deployment Repository

    Since we're using the separate repository method, let’s set up a new GitHub Pages repository for the static files.

    1. Create a GitHub Repository for GitHub Pages:

    2. Create another repository on GitHub called yourusername.github.io (replace yourusername with your GitHub username).

    3. Important: This repository name must match this pattern to be hosted properly.

    4. Clone the Deployment Repository:

    5. Clone the repository onto your computer: 

      git clone https://github.com/yourusername/yourusername.github.io.git

    6. Copy Static Files:

    7. Go to your my-knowledge-base directory and copy the built files from site/ into your deployment repository: 
      xcopy /e /i D:\git\my-knowledge-base\site\* D:\git\yourusername.github.io\

    Step 7: Commit and Push the Site to GitHub Pages

    1. Navigate to Your GitHub Pages Directory:
    cd D:\git\yourusername.github.io

    1. Add, Commit, and Push the Changes:

    2. Add all the files to Git: 

      git add .

    3. Commit the changes: 
      git commit -m "Initial MkDocs site deployment"

    4. Push to GitHub: 

      git push origin master

    5. Visit Your Website:

    6. Go to https://yourusername.github.io and you should see your website live!

    Step 8: Simple Workflow for Updates

    When you want to make changes to your MkDocs site, follow these steps:

    1. Make Edits: Edit or add new markdown files in the docs/ folder.
    2. Preview Changes Locally: Run mkdocs serve to see the changes in your browser.
    3. Build Static Site: Run mkdocs build to create the new static files.
    4. Copy Files to Deployment Repo: Copy the contents of the site/ folder to your yourusername.github.io repository.
    5. Commit and Push the updated site.

    Summary

    • MkDocs makes it easy to create a beautiful website from markdown files.
    • Two Repositories: One for the source (my-knowledge-base) and one for the deployment (yourusername.github.io).
    • Workflow: Edit > Preview > Build > Copy > Commit & Push.
